=== CHESSCOM ===
--- SUMMARY ---
Chess.com's Game Review classifies moves via an "Expected Points Model": win probability (0.00–1.00) derived from engine eval AND player rating (rating-adjusted since the Feb-2025 CAPS2 update). The official support article publishes the expected-points-LOST ladder: Best=0.00, Excellent 0–0.02, Good 0.02–0.05, Inaccuracy 0.05–0.10, Mistake 0.10–0.20, Blunder 0.20–1.00. So it is win-probability-loss based, not raw centipawn-loss. Special classifications sit outside the table: Brilliant = a good piece sacrifice where you are not bad afterwards and were not already completely winning; Great = best/near-best move that is "critical to the outcome" (only good move / turning point); Miss = failing to punish an opponent's mistake (Ø glyph). Accuracy is CAPS2 (trade secret): each move scored by category, "mostly averaged" with adjustments for mate-distance and reduced penalty for repeated blunders; book moves count as best; scores deliberately redistributed to a 50–95 "academic grading" feel; engine depth varies with player strength.

Open-source reverse-engineerings, all fetched verbatim from raw source: (1) WintrCat/freechess (the classic "Game Report", CC BY-NC-SA 4.0 — NOT code-license-friendly): pure-centipawn quadratic thresholds via getEvaluationLossThreshold(), mate-transition decision tables, Brilliant via hand-rolled isPieceHanging/getAttackers/getDefenders sacrifice detection (non-pawn non-king piece left hanging, capture not refuted by pins/mate-in-1, not winning anyway i.e. 2nd line < +700, eval ≥ 0, no promotion, not in check), Great = previous move was Blunder + top-vs-2nd-line gap ≥ 150cp + moved piece not hanging, accuracy = mean of per-classification values (blunder 0 … best 1) ×100. No Miss. (2) WintrCat/wintrchess (successor, GPL-3.0): expected points = 1/(1+e^(−0.0035·cp)); point-loss ladder Best<0.01, Excellent<0.045, Okay<0.08, Inaccuracy<0.12, Mistake<0.22, else Blunder; "Critical" (=Great) requires top move played + 2nd-best move would lose ≥0.10 expected points + not free-material capture + not having mate; Brilliant via unsafe/trapped-piece analysis; per-move accuracy 103.16·e^(−4·pointLoss)−3.17, game accuracy = plain mean per side. (3) Guilla

    "blunder": 0, "mistake": 0.2, "inaccuracy": 0.4, "good": 0.65, "excellent": 0.9,
    "best": 1, "great": 1, "brilliant": 1, "book": 1, "forced": 1

export function getEvaluationLossThreshold(classif, prevEval) {
    prevEval = Math.abs(prevEval);
    let threshold = 0;
    switch (classif) {
        case Classification.BEST:
            threshold = 0.0001 * Math.pow(prevEval, 2) + (0.0236 * prevEval) - 3.7143; break;
        case Classification.EXCELLENT:
            threshold = 0.0002 * Math.pow(prevEval, 2) + (0.1231 * prevEval) + 27.5455; break;
        case Classification.GOOD:
            threshold = 0.0002 * Math.pow(prevEval, 2) + (0.2643 * prevEval) + 60.5455; break;
        case Classification.INACCURACY:
            threshold = 0.0002 * Math.pow(prevEval, 2) + (0.3624 * prevEval) + 108.0909; break;
        case Classification.MISTAKE:
            threshold = 0.0003 * Math.pow(prevEval, 2) + (0.4027 * prevEval) + 225.8182; break;
        default: threshold = Infinity;
    }
    return Math.max(threshold, 0);
}

=== EVALBAR ===
--- SUMMARY ---
Lichess (lila) is the only major site with public eval-bar code, and it is fully extracted here. The pipeline is: (1) UCI parse layer converts side-to-move scores to white perspective; (2) a sigmoid ("winning chances") maps cp to [-1,1]; (3) the bar renders a single black div whose height% = 100 - (chances+1)*50, smoothed purely by CSS `transition: height 1s`.

The sigmoid (ui/lib/src/ceval/winningChances.ts): chances = 2/(1+exp(-0.00368208*cp)) - 1, with cp clamped to [-1000, 1000]. The constant was fit (PR #11148, scipy curve_fit) to 75k positions from 2300+ rapid games in the June 2022 lichess database, replacing the older -0.004. Lichess's published Win% (accuracy page) is the same function scaled: Win% = 50 + 50*(2/(1+exp(-0.00368208*cp))-1). Mate scores are mapped to a synthetic cp = (21 - min(10,|mate|))*100, signed — so M1→2000cp (chances 0.9987, bar 99.94% full), M10+→1100cp (0.9657, 98.3%). Because cp is clamped at 1000 (chances 0.9509, bar max 97.55% for non-mate), any forced mate always fills more than any cp eval, yet the bar is never pinned at 100% and still shrinks slightly as mate distance grows; the numeric label shows "#N". Game-over (mate 0) is filtered at parse time (`if (isMate && !povEv) return`).

Perspective: per the UCI spec, "cp [is] the score from the engine's point of view" (side to move at search root) and "mate in y moves, not plies; if the engine is getting mated use negative values". Lila converts to white-POV in protocol.ts: `const pivot = this.work.threatMode ? 0 : 1; const ev = this.work.ply % 2 === pivot ? -povEv : povEv;` (odd ply = black to move = negate; threat mode flips the pivot because a null move was injected). Board flipping never touches the math: renderGauge always computes povChances('white', ev) and adds class `reverse` when orientation is black, which CSS flips with `transform: rotateX(180deg)`. When no eval is available the gauge keeps the last value (`gaugeLast`) and dims via `.empty { opacity: 0.5 }`. Mid-search, every streamed `info` line re-renders the gauge and the 1s CSS transition absorbs depth-to-depth jumps.

// ============ 1. lichess lila: ui/lib/src/ceval/winningChances.ts (verbatim, master) ============
const toPov = (color: Color, diff: number): number => (color === 'white' ? diff : -diff);
/**
 * https://github.com/lichess-org/lila/pull/11148
 */
const rawWinningChances = (cp: number): WinningChances => {
  const MULTIPLIER = -0.00368208; // https://github.com/lichess-org/lila/pull/11148
  return 2 / (1 + Math.exp(MULTIPLIER * cp)) - 1;
};
const cpWinningChances = (cp: number): WinningChances =>
  rawWinningChances(Math.min(Math.max(-1000, cp), 1000));
const mateWinningChances = (mate: number): WinningChances => {
  const cp = (21 - Math.min(10, Math.abs(mate))) * 100;
  const signed = cp * (mate > 0 ? 1 : -1);
  return rawWinningChances(signed);
};
const evalWinningChances = (ev: EvalScore): WinningChances =>
  typeof ev.mate !== 'undefined' ? mateWinningChances(ev.mate) : cpWinningChances(ev.cp!);
// winning chances for a color: 1 infinitely winning, -1 infinitely losing
export const povChances = (color: Color, ev: EvalScore): WinningChances =>
  toPov(color, evalWinningChances(ev));
// difference in winning chances between two evaluations, in [-1, 1]
export const povDiff = (color: Color, e1: EvalScore, e2: EvalScore): number =>
  (povChances(color, e1) - povChances(color, e2)) / 2;

WinRateParams win_rate_params(const Position& pos) {
    int material = pos.count<PAWN>() + 3 * pos.count<KNIGHT>() + 3 * pos.count<BISHOP>()
                 + 5 * pos.count<ROOK>() + 9 * pos.count<QUEEN>();
    double m = std::clamp(material, 17, 78) / 58.0;
    constexpr double as[] = {-72.32565836, 185.93832038, -144.58862193, 416.44950446};
    constexpr double bs[] = {83.86794042, -136.06112997, 69.98820887, 47.62901433};
    double a = (((as[0] * m + as[1]) * m + as[2]) * m) + as[3];
    double b = (((bs[0] * m + bs[1]) * m + bs[2]) * m) + bs[3];
    return {a, b};
}
int win_rate_model(Value v, const Position& pos) {
    auto [a, b] = win_rate_params(pos);
    return int(0.5 + 1000 / (1 + std::exp((a - double(v)) / b)));   // per-mille win prob, stm POV
}

const rawWinningChances = (cp: number): WinningChances => {
  const MULTIPLIER = -0.00368208; // https://github.com/lichess-org/lila/pull/11148
  return 2 / (1 + Math.exp(MULTIPLIER * cp)) - 1;
};

  const cp = (21 - Math.min(10, Math.abs(mate))) * 100;
  return rawWinningChances(cp * (mate > 0 ? 1 : -1));
};

export const sharedWasmMemory = (lo: number, hi = 32767): WebAssembly.Memory => {
  let shrink = 4; // 32767 -> 24576 -> 16384 -> 12288 -> 8192 -> 6144 -> etc
  while (true) {
    try { return new WebAssembly.Memory({ shared: true, initial: lo, maximum: hi }); }
    catch (e) {
      if (hi <= lo || !(e instanceof RangeError)) throw e;
      hi = Math.max(lo, Math.ceil(hi - hi / shrink));
      shrink = shrink === 4 ? 3 : 4;
    }
  }
};

### lichess-puzzler tagger/util.py — foundational predicates (Python, port to chessops)
values = { PAWN: 1, KNIGHT: 3, BISHOP: 3, ROOK: 5, QUEEN: 9 }
king_values = { PAWN: 1, KNIGHT: 3, BISHOP: 3, ROOK: 5, QUEEN: 9, KING: 99 }

def material_count(board, side):
    return sum(len(board.pieces(pt, side)) * v for pt, v in values.items())
def material_diff(board, side):
    return material_count(board, side) - material_count(board, not side)

def is_defended(board, piece, square):
    if board.attackers(piece.color, square):
        return True
    for attacker in board.attackers(not piece.color, square):
        attacker_piece = board.piece_at(attacker)
        if attacker_piece.piece_type in ray_piece_types:  # B, R, Q
            bc = board.copy(stack=False)
            bc.remove_piece_at(attacker)
            if bc.attackers(piece.color, square):
                return True   # x-ray defended
    return False

def is_hanging(board, piece, square):
    return not is_defended(board, piece, square)

def can_be_taken_by_lower_piece(board, piece, square):
    for attacker_square in board.attackers(not piece.color, square):
        attacker = board.piece_at(attacker_square)
        if attacker.piece_type != chess.KING and values[attacker.piece_type] < values[piece.piece_type]:
            return True
    return False

def is_in_bad_spot(board, square):
    piece = board.piece_at(square)
    return (bool(board.attackers(not piece.color, square)) and
            (is_hanging(board, piece, square) or can_be_taken_by_lower_piece(board, piece, square)))

def is_trapped(board, square):
    if board.is_check() or board.is_pinned(board.turn, square): return False
    piece = board.piece_at(square)
    if piece.piece_type in [PAWN, KING]: return False
    if not is_in_bad_spot(board, square): return False
    for escape in board.legal_moves:
        if escape.from_square == square:
            capturing = board.piece_at(escape.to_square)
            if capturing and values[capturing.piece_type] >= values[piece.piece_type]:
                return False
            board.push(escape)
            if not is_in_bad_spot(board, escape.to_square):
                return False
            board.pop()
    return True

### lichess-puzzler tagger/cook.py — motif detectors (operate on the PV/mainline)
def fork(puzzle):
    for node in puzzle.mainline[1::2][:-1]:
        if util.moved_piece_type(node) is not KING:
            board = node.board()
            if util.is_in_bad_spot(board, node.move.to_square): continue
            nb = 0
            for piece, square in util.attacked_opponent_squares(board, node.move.to_square, puzzle.pov):
                if piece.piece_type == PAWN: continue
                if util.king_values[piece.piece_type] > util.king_values[util.moved_piece_type(node)] or (
                    util.is_hanging(board, piece, square)
                    and square not in board.attackers(not puzzle.pov, node.move.to_square)):
                    nb += 1
            if nb > 1: return True
    return False

def sacrifice(puzzle):
    diffs = [material_diff(n.board(), puzzle.pov) for n in puzzle.mainline]
    initial = diffs[0]
    for d in diffs[1::2][1:]:
        if d - initial <= -2:
            return not any(n.move.promotion for n in puzzle.mainline[::2][1:])
    return False

def discovered_check(puzzle):
    for node in puzzle.mainline[1::2]:
        board = node.board()
        checkers = board.checkers()
        if checkers and not node.move.to_square in checkers:
            return True
    return False

def skewer(puzzle):
    for node in puzzle.mainline[1::2][1:]:
        prev = node.parent
        capture = prev.board().piece_at(node.move.to_square)
